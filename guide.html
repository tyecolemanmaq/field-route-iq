Before the session · one-time setup (5 min)

Prerequisites

Do this before you arrive. A missing or wrong login is the single thing that'll cost you time on the day — don't sort it out during the race.

✓

Node.js 20+

node --version # want v20 or newer
✓

Install the GitHub Copilot CLI

npm install -g @github/copilot

On Windows PowerShell, run this once so the CLI is allowed to run (fixes the "running scripts is disabled" error):

Set-ExecutionPolicy -Scope CurrentUser RemoteSigned -Force

Then check copilot --version. If it's "not recognized," reopen your terminal; if still not found, add npm to PATH and reopen:

[Environment]::SetEnvironmentVariable('Path',[Environment]::GetEnvironmentVariable('Path','User')+';'+"$env:AppData\npm",'User')
✓

Sign in — with the RIGHT account

copilot

Sign in with the GitHub account that has your Copilot license (your work account) — not a personal one. Wrong account = no models available, and nothing will run.

✓

Confirm it works

copilot -p "hello"

You should see an AI Credits … line at the end. No line = you're not signed in with a Copilot-enabled account — fix that now.

The flow · six steps

What you actually do

1

Fork the repo

On the repo page, click Fork. That's your private copy of the arena — no invite needed.

2

Clone your fork & install

git clone https://github.com/<you>/field-route-iq && cd field-route-iq && npm install
3

Build your harness — this is the whole competition

Engineer whatever context makes the agent understand this brownfield project and build the feature correctly in as few credits as possible. How you do that is the challenge, and where the leaderboard is won.

SPEC.md is the only source of truth. Read it closely — not everything else in this repo is reliable.

4

Run the agent — it builds src/pricing/engine.ts

Run the agent through the wrapper (details below). It uses your harness files, writes only that one file, and records the run's cost to COST.txt automatically.

node agent-run.mjs --model gpt-5-mini
5

Iterate — and stay lean

Refine your harness, maybe switch models, re-run. Each run adds to your cumulative COST.txt — so a lean harness that lands it in one cheap run beats a pile of retries. The agent must never write or run tests.

6

Commit & push — then re-push to climb

git add src/pricing/engine.ts COST.txt && git commit -m submit && git push

At each checkpoint the leaderboard updates. Refine your harness, maybe switch models, update COST.txt, push again. Pencils down = your last push.

How cost is measured — do this

Run your agent through the Copilot CLI

The Copilot CLI prints the exact credits it spent after every run. We ship a small wrapper that runs it (using your harness files automatically) and adds each run's credits to COST.txt for you — nothing to read off a screen.

Pick your model with --model <name>. To see your available models, run copilot then /model. Cheaper models cost fewer credits — e.g. gpt-5-mini is far cheaper than gpt-5.3-codex (the strong default). The game is whether your harness can carry the cheapest model that still passes the gate.

One-time setup: npm install -g @github/copilot, then run copilot and sign in. Credits already price in the model, so a cheaper model = fewer credits. Top scorers screen-share their credit total before prizes. Under-reporting is a DQ.

How you're scored

Qualify, then cheapest wins

The rules that matter

Read these once

  • You build the harness; the agent writes the code. No hand-writing or hand-editing anything under src/.
  • Scored deliverable = priceOrder only (SPEC §1.1). The OrderPage/UI wiring in the spec is not scored — skip it; don't spend credits on it.
  • The agent writes no tests and never scores itself. There are none in the repo; the judge runs the hidden suite at each checkpoint.
  • SPEC.md is the only truth. Trust it over any file or comment that disagrees.
  • All spend counts — exploration, failed runs, everything. Report your cumulative credits honestly.
  • Copying another fork's engine = DQ. The judge flags identical engines. (Harness files can stay local; only engine.ts + COST.txt need pushing.)
